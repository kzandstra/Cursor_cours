# Product Brief: Evariste Price & Quote Assistant (Assistant Prix Devis)

## 1. Executive Summary
The **Evariste Price & Quote Assistant** is a modern, AI-powered conversational web application tailored for the Public Works (Travaux Publics) sector. Its primary goal is to provide field workers, site managers, and buyers with an instant, frictionless way to find material costs (sand, concrete, asphalt, etc.) and automate the heavy lifting of data entry when dealing with new supplier quotes (devis). 

The tool replaces manual PDF data transcription with an automated pipeline that extracts pricing and maps it directly into a central database, making it immediately available for search via a hybrid chatbot interface.

---

## 2. Core Objectives & Value Proposition
- **Frictionless Data Retrieval:** Enable non-technical users to query standard prices simply by asking a question or clicking a quick-action button in a chat interface.
- **Zero-Touch Data Entry:** Fully automated extraction of key data points (Supplier, Product Name, Unit Price, Tax, Quantities) from uploaded PDF quotes using best-in-class AI (Claude 3.5 Sonnet).
- **Centralized Source of Truth:** Unify unstructured documents into a structured PostgreSQL database (Supabase), eliminating scattered files and outdated pricing sheets.
- **Universal Accessibility:** Designed as an embeddable, responsive web module meant to live directly inside the company’s SharePoint/Intranet.

---

## 3. Target Audience
- **Conducteurs de Travaux (Site Managers):** Need quick material estimates for their daily site operations without opening complex ERP systems.
- **Acheteurs (Buyers / Procurement):** Need to compare current material base prices against new incoming quotes.
- **Administratifs (Office Teams):** Need an automated way to process dozens of supplier quotes per week without manual data entry.

---

## 4. Key Features
### A. Hybrid Chat Interface
Leveraging both natural language search and quick-action chips. When a user searches for a product (e.g., "Prix du sable 0/4"), the system queries the Supabase database.
- **Smart Data Joining:** The search query compares the core catalog (`produits`) with the latest offers (`offres_prix`) and returns dynamic, beautifully formatted Product Cards showing the location, product name, and best active price.
- **Semantic Filtering:** Auto-categorization of keywords (e.g., "grave", "gnt", "béton") for robust matching using `.ilike` PostgreSQL functions.

### B. Intelligent PDF Upload (AI Pipeline) 
Users can click "Téléverser un devis" to upload a raw PDF. No manual form-filling is required.
- **Direct Hand-off:** The frontend bypasses standard storage procedures and securely POSTs the binary PDF to an **n8n Webhook**.
- **AI Extraction:** The n8n workflow routes the PDF to an Anthropic LLM (Claude 3.5 Sonnet) tailored with a system prompt to perform advanced OCR and semantic extraction.
- **Auto-Insertion:** The JSON payload generated by the AI automatically populates the Supabase tables, making the new pricing immediately searchable by the chatbot.

### C. Security & Authentication
- **Role-Based Access Control (RBAC):** Users authenticate via Supabase Auth (Email/Password or Google OAuth). An `access` table verifies if the user is `admin`, `user`, or `readonly`.
- **Permission Guards:** `readonly` users can browse prices but the PDF upload function is disabled to prevent database flooding.

---

## 5. Technical Architecture Stack
The platform is built on a composable, modern stack ensuring speed and low maintenance:

| Component | Technology | Role |
| :--- | :--- | :--- |
| **Frontend** | React, ViteJS, TypeScript | Provides the chat UI, renders the Product Cards, handles auth bridging, and manages file inputs. |
| **Database & Auth** | Supabase (PostgreSQL) | Handles User Sessions, stores structured data (`produits`, `offres_prix`), and provides REST APIs for querying. |
| **Automation Flow** | n8n | The orchestration engine connecting the Next.js/React frontend with the Language Model and the Database. Listens to Webhooks and Google Drive. |
| **Intelligence** | Anthropic Claude 3.5 Sonnet | Receives PDF binaries from n8n, extracts unstructured text, and formats it reliably into structured JSON formats. |
| **UI/UX Styling** | Vanilla CSS, Lucide Icons | Custom styling following a Pale Blue and Orange brand identity to match the Public Works aesthetic. |

## 6. Workflow Diagrams
### The Upload & Extraction Flow:
`[User uploads PDF in React UI]` 
  **→** `[POST /webhook/upload-devis-ia to n8n]` 
  **→** `[n8n reads Binary Data]` 
  **→** `[Anthropic Claude reads PDF and outputs structured JSON]` 
  **→** `[n8n Inserts Row into Supabase 'produits']`
  **→** `[New Price is Live in Chat]`

### The Search Flow:
`[User types "Béton C25/30"]` 
  **→** `[React filters keywords]` 
  **→** `[Supabase queries 'produits' LEFT JOIN 'offres_prix']` 
  **→** `[React resolves best price]` 
  **→** `[UI renders Product Card in chat]`

---

## 7. Future Perspectives & Roadmap
- **Real-time Vector Search (RAG):** Transitioning from `.ilike` matching to actual vector embeddings using `pgvector` inside Supabase to handle complex conversational queries (e.g., "Quel fournisseur est le moins cher pour du sable près de Lyon ?").
- **Voice Integration:** Allowing site managers to tap a microphone button and ask questions instead of typing while on the construction site.
- **Analytics Dashboard:** A specific admin view aggregating all price trends mapped over time from the extracted quotes.
